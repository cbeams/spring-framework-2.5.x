This package contains an extremely efficient and complete implementation of JMS selectors based on the JMS Specifications 1.1. It uses a JavaCC generated parser to parse the selector string. An in-memory representation of the selector is built during the parse. Once a selector has been parsed, its in-memory representation can be evaluated with minimal overhead. As a result, selector evaluations are extremely efficient at run-time. As a comparison, this implementation is roughly 4 times faster than implementation found in JBOSS.

In order to use this package do the following:

Development Steps
1 Create an instance of Selector by using one of its static factory methods.
2 Get the list of identifiers encountered during the parse by calling getIdentifiers() on the Selector instance.
3 For each Object that you want to evaluate with the selector create a Map and populate it with the identifier names and values. Or, create an instance of IValueProvider.
4 Call eval() on the Selector instance to evaluate the selector. You can invoke either the Map or the IValueProvider version of the eval() method. Note, if you use the IValueProvider version of eval() then callbacks on the IValueProvider instance will be invoked to get values of identifiers during evaluation.
